# CAP — 360° Captures

Room-based panoramic captures with spatial and temporal filtering.

## What the user sees

- **Room list** with thumbnails, capture count per room, and latest capture date.
- **Level metadata** (e.g. "LEVEL 01") shown per room.
- **Filtering** by date range, building level, and room name.
- **Multi-select** support for rooms.
- **Detail panel** opens on room selection — slideshow viewer, keyboard navigation, ZIP download.
- **Capture point names** when available from the room capture API.

## Data flow

```
360 Captures API v2  →  paginated, 1000/page
  loaded into DuckDB as `captures_360` table
  (contains URLs, spatial coordinates xMeters/yMeters/zMeters, timestamps)

Room metadata parquets  →  `project-rooms` + `project-levels` DuckDB tables
  (roomId → roomName → levelName → elevation)

Dashboard360Service
  ├─ lazy: only initializes when CAP tab is first opened
  ├─ room metadata loaded eagerly (for filter panel population before tab open)
  ├─ subscribes to elementVisibility$ (fires after PRG materialization)
  └─ emits: roomSummaries$, selectedCaptures$
```

## How filtering works

Filters are applied at the **capture level** first, then results are grouped by room:

```
SELECT modelRoomId, COUNT(*), MAX(imageTakenOn), first capture URL
FROM captures_360
WHERE imageTakenOn BETWEEN :startDate AND :endDate
  AND levelName IN (:selectedLevels)   -- if level filter active
GROUP BY modelRoomId
```

Rooms without captures matching the current filter disappear from the list. Counts reflect the filtered subset, not the total.

Selecting a room sets an `imageId` filter in `DashboardFilterService`. This selection filter doesn't trigger a data re-query — it only controls which room's detail panel is shown and which pinpoints are highlighted in the 3D viewer.

## Pinpoints in the viewer

Each 360 capture has spatial coordinates (xMeters, yMeters, zMeters) from its `modelRoomId`. The 3D viewer renders these as clickable pinpoints on the model. Clicking a pinpoint opens the CAP detail panel for that room.

## Key service

`Dashboard360Service` in `services/dashboard-360/`. Lazy — only initializes on first CAP tab open. Room/level metadata is loaded before the service fully initializes so the filter panel (Level dropdown) is populated immediately.

## Implementation note

The room list currently reuses the `IssueTable` component from QLT with adapted data. Changes to `IssueTable` can unintentionally affect the CAP room list — see [pitfalls.md](pitfalls.md).
